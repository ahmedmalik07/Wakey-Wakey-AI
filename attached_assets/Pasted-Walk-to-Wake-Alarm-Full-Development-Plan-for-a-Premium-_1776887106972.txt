Walk-to-Wake Alarm: Full Development Plan for a Premium, Zero-Cost Android App
This is a complete, production-ready blueprint to build Walk-to-Wake — an alarm that refuses to shut up until the phone’s pedometer confirms you’ve walked at least 30 steps. The app will feel premium (think Sleep as Android or Alarmy but focused, modern, and actually free), while staying 100% within free tiers and open-source tools. No paid APIs, no cloud bills, no surprises.
I’ve thought deeply about reliability (Android’s aggressive battery/Doze restrictions), user delight, AI that actually adds value without eating quotas, and scalability. The result is a feature-rich app that could easily go viral on r/androidapps or F-Droid.
1. Core Feature (Non-Negotiable)

Alarm triggers → full-screen overlay + loud looping sound (user-chosen ringtone or custom MP3).
Sound does not stop (even if user presses power button or app is backgrounded) until real walking steps are detected.
Uses Health Connect (preferred) or fallback to SensorManager.TYPE_STEP_COUNTER.
Records baseline steps at alarm start → dismisses when Δsteps ≥ user-configurable threshold (default 30).

2. Feature Set – Making It Far From Basic
Alarm Management

Recurring alarms (Mon–Sun, custom days, weekends only).
Labels, custom volume ramp-up (0% → 100% over 30s), vibration patterns.
“Gentle wake” mode: starts silent + gradually louder.
Snooze with step requirement (optional toggle).

Walk-to-Wake Enhancements

Configurable step goal (10–100) per alarm.
“Proof of walking” progress ring + live step counter.
Fallback mode (if no pedometer): accelerometer-based shake detection (simple algorithm) + warning banner.

Statistics & Insights Dashboard

Success rate, average time-to-30-steps, weekly streaks.
Beautiful charts (Vico Compose library – free).
Export to CSV.

Health Connect Integration

Automatic step syncing (Android 14+ built-in on-device counting).
Reads/writes steps for accuracy and battery efficiency.
Optional daily step goal linking.

AI-Powered Features (All Free Tier / On-Device)
These are the “wow” factors that make the app feel intelligent without cost:

AI Morning Motivator (Gemini 2.5 Flash-Lite via official Android SDK)
After successful dismissal: generates personalized 1–2 sentence motivation + fun fact or joke.
Prompt example: “User just walked 30 steps to kill their 6:45 AM alarm. Generate a short, energetic, positive message mentioning their name and today’s weather if available.”
Runs on-device where Gemini Nano is available (Pixel/Samsung); falls back to cloud free tier (15 RPM / 1,000 RPD – more than enough; 1 call per alarm max).

Smart Alarm Advisor (on-device or Gemini API)
Analyzes past 7–30 days of alarm data (stored in Room).
Suggests: “You succeed 92% when waking at 7:00 AM on weekdays. Want me to shift your Friday alarm 15 min later?”
Uses simple TensorFlow Lite model (pre-trained lightweight regressor) + Gemini for natural-language summary.

Voice Command Setup (Android SpeechRecognizer – fully on-device)
“Hey, set a walk-to-wake alarm for 7 AM weekdays with 40 steps.”
Voice confirmation readout via Text-to-Speech.

Weekly AI Insight Report
Every Sunday: “You beat 6/7 alarms this week! Your fastest walk time was 18 seconds on Tuesday. Here’s how to improve sleep consistency…”


Polish & Delight

Material You dynamic theming + dark mode.
Home-screen widget for quick alarm toggle.
Achievements & streaks (e.g., “30-Day Early Riser”).
Accessibility-first (TalkBack, large touch targets, high-contrast mode).
Backup/restore alarms via local JSON export (no cloud needed).

3. Technical Architecture (Modern, Maintainable, Free)

Language: Kotlin 100%
UI: Jetpack Compose + Material 3 (no XML)
Architecture: MVVM + Clean Architecture + Hilt (DI)
Data: Room DB + DataStore (preferences)
Background:
AlarmManager + BroadcastReceiver for scheduling
Foreground Service (AlarmRingingService) for sound + step monitoring (highest priority, user-visible notification “Alarm ringing – walk to dismiss”)

Steps:
Primary: Health Connect SDK (androidx.health.connect:connect-client)
Fallback: SensorManager + TYPE_STEP_COUNTER

AI:
Gemini SDK (com.google.ai.client.generativeai:generativeai)
On-device: Gemini Nano (via Firebase AI Logic where supported) or ML Kit + custom TFLite

Audio: MediaPlayer + AudioAttributes.USAGE_ALARM
Charts: Vico Compose (lightweight, free)
Testing: JUnit5 + Compose UI testing + Robolectric

Key Permissions (all declared + runtime requested):

ACTIVITY_RECOGNITION
POST_NOTIFICATIONS
SCHEDULE_EXACT_ALARM (Android 12+)
Health Connect READ_STEPS + WRITE_STEPS
Optional: FOREGROUND_SERVICE + battery optimization exemption prompt
